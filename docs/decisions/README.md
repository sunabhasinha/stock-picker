# Decisions (L3 — memory)

Architecture Decision Records: WHY things are the way they are. Append-only —
a reversed decision gets a NEW record marked "supersedes ADR-XXXX", never an
edit. This is what lets an agent (or a new dev) answer "why is it like this?"
without archaeology.

What earns an ADR: cross-module choices, dependency additions, anything a
future contributor would plausibly try to "fix" without knowing the reason.
Journal judgment-calls get promoted here once they prove out (weekly review).

Format: Context -> Decision -> Consequences. One page max. Numbered.
